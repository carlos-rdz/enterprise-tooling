🟠 MAJOR — Eligibility uses OR instead of AND — approves applicants who meet only one of three required criteria
  File:     flexpay/eligibility.py
  Issue:    The PR description states Standard tier requires tenure AND clean delinquency AND FICO floor ("18+ months on the card, no >30dpd in last 12mo, FICO at-or-above the model floor"). The implementation joins the three predicates with `or`, so a brand-new account with a recent 90dpd would be approved as long as FICO ≥ 680 (and vice versa). This is the opposite of "tightening" eligibility and is a UDAAP/credit-risk correctness violation.
  Evidence: return has_tenure or clean_delinquency or fico_ok
  Fix:      Change to `return has_tenure and clean_delinquency and fico_ok`. Add a unit test that asserts a member failing any one criterion returns False.
